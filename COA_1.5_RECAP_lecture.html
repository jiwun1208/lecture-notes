디지털 시스템의 데이터 표현 (1)

디지털 시스템은 연속적인 전압을 이산적인 값(0과 1)으로 구분하여 데이터를 처리한다. 이것이 바로 이진(binary) 체계의 핵심이다.
상세 설명

디지털 시스템은 이산 데이터(discrete data)를 다룬다. 물리적으로는 전기 전압(voltage)을 사용하는데, 이 전압을 두 구간으로 나누어 해석한다:

  • 높은 전압 구간(High voltage interval) → 논리 1
  • 낮은 전압 구간(Low voltage interval) → 논리 0

이렇게 전압을 두 레벨로 나누는 것이 이진법(binary)의 물리적 기반이다. 이진 숫자(binary digit, 줄여서 bit)는 다양한 방식으로 표현할 수 있다:

  • 0과 1
  • T(True)와 F(False)
  • H(High)와 L(Low)

특수한 용도로는 3-레벨(ternary)이나 4-레벨(quad-level) 숫자를 사용하기도 한다. 이들은 하나의 신호선에 더 많은 정보를 실을 수 있어 대역폭 효율(bandwidth efficiency)이 높다.

핵심 용어
영어한국어설명

Logic Level논리 레벨전압을 구분하는 기준 (High/Low)
시험 포인트: "왜 이진법을 사용하는가?" — 전압 구간을 2개로 나누면 잡음(noise)에 대한 여유가 커져 오류가 적다. 레벨이 많아질수록 구간 간격이 좁아져 오류 가능성이 높아진다.

디지털 시스템의 데이터 표현 (2)

현실 세계의 연속적인 데이터를 디지털 시스템에서 처리하려면 양자화(Quantization)/디지털화(Digitization) 과정이 필요하며, 이를 수행하는 장치가 ADC이다.
상세 설명

현실의 데이터(온도, 소리, 빛 등)는 연속적(continuous)이다. 이를 디지털 시스템에서 처리하려면 이산적(discrete)인 값으로 변환해야 한다. 이 과정을 양자화(quantization) 또는 디지털화(digitization)라고 한다.

아날로그-디지털 변환기(ADC, Analog-Digital Converter)는 아날로그 세계와 디지털 세계의 인터페이스 역할을 한다:

  • 아날로그 신호를 주기적으로 샘플링(sampling, 수집)한다
  • 클록 신호(clock signal)에 맞춰 디지털화된 데이터를 출력한다
  • 예를 들어, 4-bit ADC는 아날로그 신호를 0000~1111(0~15)의 16단계로 변환한다

슬라이드의 다이어그램은 왼쪽에 연속적인 아날로그 파형이 있고, ADC를 거쳐 오른쪽에 계단 형태의 디지털 데이터(4비트 값)가 클록 주기마다 출력되는 모습을 보여준다.

디지털 시스템과 분류

디지털 시스템(회로)은 논리 게이트들의 조합으로 이루어지며, 크게 조합 논리(Combinational Logic)와 순차 논리(Sequential Logic) 두 가지로 분류된다.
상세 설명

디지털 시스템(또는 디지털 회로, 디지털 컴포넌트)은 수많은 논리 게이트(logic gate)들로 구성된다. 이러한 디지털 회로는 체계적으로 두 종류로 분류할 수 있다:

  • 조합 논리(Combinational Logic): 출력이 오직 현재 입력에 의해서만 결정된다. 과거의 상태나 이력과 무관하다. 예: 덧셈기, 멀티플렉서
  • 순차 논리(Sequential Logic): 출력이 현재 입력뿐만 아니라 현재의 논리 상태(state)에 의해서도 결정된다. 즉, "기억"이 있는 회로이다. 예: 카운터, 레지스터

비유하자면, 조합 논리는 "계산기"(버튼을 누르면 바로 답이 나옴)이고, 순차 논리는 "자판기"(현재 투입된 금액 상태에 따라 동작이 달라짐)와 같다.

핵심: 조합 논리 = 입력만으로 출력 결정, 순차 논리 = 입력 + 현재 상태로 출력 결정. 이 구분이 디지털 회로 설계의 가장 기본적인 분류이다.

디지털 컴포넌트의 신호(Signals)

디지털 컴포넌트들은 신호(signal)를 통해 서로 통신하며, 각 신호는 1비트 또는 여러 비트로 구성될 수 있다.
상세 설명

디지털 컴포넌트는 다른 컴포넌트들과 정보를 주고받는다. 이때 사용되는 통신 매체가 바로 신호(signal)이다.

  • 신호는 1비트일 수도 있고, 여러 비트/핀으로 구성될 수도 있다
  • 좋은 신호 이름은 직관적이어야 한다: GO, PAUSE, READY, ERROR 등

슬라이드의 MEMORY 컴포넌트 예시를 보면:

  • write_address, read_address: 메모리의 읽기/쓰기 주소를 지정하는 신호 (여러 비트)
  • write_data, read_data: 실제 데이터를 전달하는 신호 (여러 비트)
  • ENABLE, RESET: 메모리의 동작을 제어하는 신호 (1비트)
  • VALID: 출력 데이터가 유효한지 알려주는 신호 (1비트)

신호의 활성 레벨 (Active Levels)

신호가 자신의 이름이 의미하는 동작을 수행하는 상태를 활성(active)이라 하며, Active-High와 Active-Low 두 가지 방식이 있다.
상세 설명

신호가 활성(active) 상태라 함은, 그 신호가 자기 이름이 뜻하는 기능을 실제로 수행하고 있다는 의미이다. 예를 들어 ENABLE이라는 신호가 활성이면, 연결된 장치가 실제로 동작 가능 상태가 된다.

활성 시점의 두 가지 방식:

  • Active-High: 신호가 1(High)일 때 활성. 가장 직관적인 방식
  • Active-Low: 신호가 0(Low)일 때 활성. 이름 뒤에 특수 표기를 붙인다

신호의 상태:

  • Asserted(어서트): 신호가 활성 상태에 있음
  • De-asserted(디어서트): 신호가 비활성 상태에 있음

Active-Low 신호의 표기법: SIGNAL~, ~SIGNAL, /SIGNAL, SIGNAL_L, SIGNAL_N 등 다양한 방법으로 Active-Low임을 나타낸다.

주의: Active-Low 신호는 0이 "켜진" 상태이다. RESET_N이라는 신호가 0이면 리셋이 동작 중이라는 뜻이다. 초보자가 자주 혼동하는 부분이므로 표기법을 잘 확인해야 한다.

논리 게이트 — 조합 논리의 기본 구성 요소

논리 게이트(Logic Gate)는 디지털 시스템의 가장 기본적인 구성 블록으로, 실제 물리 회로(CMOS 트랜지스터)의 추상화이다.
상세 설명

논리 게이트는 디지털 시스템을 구성하는 기본 빌딩 블록이다. 현대 디지털 시스템은 근본적으로 수십억 개의 논리 게이트로 구성된다.

논리 게이트는 디지털 세계를 위한 추상화 레이어(abstraction layer)이다:

  • 논리 게이트와 실제 물리적 회로 사이에 대응 관계가 존재한다
  • 실제로는 상보적 트랜지스터 쌍(CMOS)으로 구성된다 — PMOS와 NMOS의 쌍

슬라이드에서 보여주는 인버터(NOT 게이트) 회로를 예로 들면:

  • 위쪽의 PMOS 트랜지스터와 아래쪽의 NMOS 트랜지스터가 직렬로 연결
  • 입력(VIN)이 0이면 PMOS가 켜지고 NMOS가 꺼져 출력(VOUT)이 1
  • 입력(VIN)이 1이면 PMOS가 꺼지고 NMOS가 켜져 출력(VOUT)이 0

NOT 게이트 (인버터, Inverter)

NOT 게이트는 유일하게 입력 1개, 출력 1개를 가지는 게이트로, 입력값을 반전(invert)시킨다.
상세 설명

NOT 게이트(인버터)는 가장 단순한 논리 게이트이다.

  • 대수적 표현: A' (A의 보수)
  • 다양한 표기법: A', ~A, A-bar 등
  • 동작: 입력이 0이면 출력 1, 입력이 1이면 출력 0

AND 게이트 / NAND 게이트

AND는 모든 입력이 1일 때만 출력이 1이고, NAND는 AND의 출력을 반전한 것이다.
상세 설명

대수적 표현:

  • AND: \( O = A \cdot B \)
  • NAND: \( O = (A \cdot B)' \)

AND 게이트는 두 입력 모두 1일 때만 출력이 1이다. NAND 게이트는 AND의 출력에 반전 버블(inversion bubble)이 붙어 NOT을 나타낸다.

XOR 게이트 (Exclusive OR)

XOR 게이트는 두 입력이 서로 다를 때 1을 출력한다. 짝수 개의 입력이 1일 때 출력이 1이다.
상세 설명

대수적 표현:

  • \( O = A \oplus B \)
  • \( O = A \cdot B' + A' \cdot B \) (NOT, AND, OR 게이트로 분해 가능)

XOR은 "배타적 OR"라는 뜻으로, 두 입력 값이 서로 다를 때만 1을 출력한다.

XOR은 덧셈기(adder)의 핵심 구성 요소이며, 패리티(parity) 검사, 암호화 등에 광범위하게 사용된다.

이진 디코더 (Binary Decoder)

이진 디코더는 n비트 이진 입력을 \(2^n\)개 출력 코드로 변환하며, 원-핫 인코딩(one-hot encoding)으로의 변환에 주로 사용된다.
상세 설명

이진 디코더(Binary Decoder)는 n비트의 이진 입력을 받아 \(2^n\)개의 출력 중 정확히 하나만 활성화시키는 조합 논리 회로이다.

주요 용도:

  • 원-핫(one-hot) 변환: 이진 숫자를 "해당 위치만 1인" 형태로 변환
  • 명령어 디코딩(instruction decoding): CPU에서 명령어를 해석할 때 사용
  • 메모리 모듈 선택: 특정 메모리 칩을 활성화할 때 사용

예를 들어, 2비트 입력 "10"(=2)이 들어오면, 4개 출력 중 3번째(인덱스 2) 출력만 1이 되고 나머지는 0이 된다.

멀티플렉서 (Multiplexer, MUX)

멀티플렉서(MUX)는 여러 입력 중 하나를 선택하여 출력으로 내보내는 회로로, 디지털 시스템에서 가장 많이 사용되는 컴포넌트 중 하나이다.
상세 설명

멀티플렉서(MUX)는 여러 개의 데이터 입력 중에서 선택 신호(selection signal)에 의해 지정된 하나를 골라 출력하는 조합 논리 회로이다.

  • s비트의 선택 입력이 있으면, 최대 \(2^s\)개의 데이터 입력을 가질 수 있다
  • 내부적으로 이진 디코더가 한 번에 하나의 스위치만 켜는 방식으로 동작한다
  • 각 데이터 입력은 여러 비트(multi-bit)일 수 있다

비유하자면, MUX는 TV 채널 선택기와 같다. 여러 채널(입력)이 있고, 리모컨(선택 신호)으로 하나의 채널을 골라 화면(출력)에 보여주는 것이다.

MUX의 응용: DeMUX와 Bus

DeMUX는 MUX의 역동작(1-to-m)이고, Bus는 여러 소스와 목적지를 연결하는 공유 통신 경로이다.
상세 설명

디멀티플렉서(DeMUX):

  • MUX가 m-to-1(다대일)이라면, DeMUX는 1-to-m(일대다)이다
  • 하나의 입력 데이터를 지정된 목적지 중 하나로 전송하는 데 사용된다

버스(Bus):

  • 여러 소스 모듈과 목적지 모듈을 상호 연결하는 컴포넌트
  • 임의의 소스에서 임의의 목적지로 데이터를 전송할 수 있다
  • 내부적으로 MUX와 DeMUX의 조합으로 구현된다

전파 지연 (Propagation Delay)

실제 논리 게이트는 결과를 즉시 출력하지 않는다. 출력이 안정되기까지 걸리는 시간을 전파 지연(propagation delay, tpd)이라 한다.
상세 설명

"논리 게이트가 결과를 즉시 출력하는가?" — 아니다!

  • 논리 게이트는 출력값이 안정화되기까지 시간이 필요하다. 이를 전파 지연(propagation delay, tpd)이라 한다
  • 지연은 여러 변수에 의해 결정된다: 상태(state), 전압(voltage), 온도(temperature), 공정(process) 등
  • 디지털 시스템의 지연 시간(latency)은 근본적으로 게이트들의 전파 지연에서 비롯된다

전파 지연은 게이트 하나하나에서 발생하며, 여러 게이트를 직렬로 연결하면 각 게이트의 지연이 누적된다. 이것이 회로의 최대 동작 속도(maximum operating frequency)를 결정하는 핵심 요소이다.

핵심: 회로의 최대 동작 주파수는 가장 긴 경로(critical path)의 전파 지연에 의해 제한된다. 게이트가 많이 직렬 연결될수록 지연이 커지고 속도가 느려진다.

순차 논리에서의 상태 (States)

복잡한 디지털 시스템은 서로 다른 입력에 대해 다른 동작을 보일 수 있으며, 이러한 동작은 상태(state)에 의해 결정된다. 상태는 유한 상태 기계(FSM)로 구현된다.
상세 설명
  • 복잡한 디지털 시스템은 서로 다른 입력에 대해 서로 다른 행동을 보일 수 있다
  • 순차 회로는 서로 다른 상태(state)에서 다르게 행동할 수 있다
  • 상태는 "상태 메모리(state memory)"에 저장된다
  • 순차 논리는 유한 상태 기계(Finite State Machine, FSM)로 구현된다

비유하자면, 자판기의 동작을 생각해보자. "대기" 상태에서 동전을 넣으면 "투입" 상태로 전이하고, 버튼을 누르면 "배출" 상태가 된다. 같은 버튼이라도 현재 상태(돈이 들어왔는지)에 따라 결과가 다르다.

D 플립플롭 (D Flip-Flop)

D 플립플롭은 클록 신호의 에지(edge)에 민감하게 반응하여 데이터를 전파하는 기본 순차 소자이다.
상세 설명

D 플립플롭(D Flip-Flop)은 순차 논리의 가장 기본적인 저장 소자이다.

  • 에지 감응형(edge-sensitive) 컴포넌트 — 클록 신호의 상승 에지(rising edge) 또는 하강 에지(falling edge)에서만 동작한다
  • 주기적인 클록 신호(CLK)가 플립플롭에 공급된다
  • CLK은 디지털 시스템의 "심장 박동(heartbeat)"으로 비유된다
  • CLK의 상승/하강 에지에서 데이터 비트(D)를 출력(Q)으로 전파한다

쉽게 말해, D 플립플롭은 "클록이 째깍할 때마다 D 입력의 값을 Q 출력으로 복사하는" 소자이다. 클록 에지 사이에서는 Q 값이 변하지 않는다.

플립플롭을 이용한 상태 변수 저장

플립플롭은 클록 에지 타이밍에서만 출력이 갱신되므로, 다음 클록 에지까지 현재 상태를 유지할 수 있다.
상세 설명

플립플롭의 핵심 특성:

  • 출력(Q) 업데이트는 오직 클록 신호의 에지 타이밍에서만 트리거된다
  • CLK의 양(positive) 또는 음(negative) 에지가 올 때까지 이전(=현재) 상태를 출력(Q)에 유지한다

이것이 바로 "기억" 기능이다. 조합 논리만으로는 값을 기억할 수 없지만, 플립플롭은 클록 사이에 값을 안정적으로 보존함으로써 상태 변수(state variable)를 저장할 수 있다.

D 플립플롭의 타이밍

플립플롭은 조합 논리의 입출력을 묶어주며, 셋업 시간(tSETUP)과 홀드 시간(tHOLD)을 반드시 만족해야 한다.
상세 설명

플립플롭은 조합 논리의 입출력 사이에 위치하여 데이터 흐름을 동기화한다. 그런데 플립플롭이 정상적으로 동작하려면 두 가지 타이밍 조건을 반드시 만족해야 한다:

  • tSETUP (셋업 시간): 조합 논리의 결과가 클록 에지 이전에 도착해야 하는 최소 시간. 즉, 데이터가 충분히 일찍 안정화되어야 한다.
  • tHOLD (홀드 시간): 조합 논리의 결과가 클록 에지 이후에도 안정적으로 유지되어야 하는 최소 시간.

이 두 조건을 만족하지 못하면, 오동작(malfunction)이 발생할 확률이 매우 높다. 이를 타이밍 위반(timing violation)이라 하며, 실제 칩 설계에서 가장 중요한 검증 항목 중 하나이다.

주의: tSETUP과 tHOLD를 위반하면 메타안정 상태(metastability)에 빠질 수 있다. 출력이 0도 1도 아닌 불확정 상태가 되어 전체 시스템이 오동작할 수 있다.

레지스터 (Registers)

레지스터는 여러 개의 플립플롭을 묶어 다중 비트(multi-bit) 데이터를 저장하는 컴포넌트이다.
상세 설명

레지스터(Register)는 플립플롭들의 집합이다:

  • 각 플립플롭이 데이터의 1비트에 해당한다
  • 따라서 n비트 레지스터는 n개의 플립플롭으로 구성된다
  • 다중 비트 상태 변수(multi-bit state variable)를 보존하는 저장 컴포넌트이다

슬라이드의 간단한 마이크로프로세서 예시:

  • 피연산자 A를 위한 레지스터(Register for operand A)
  • 피연산자 B를 위한 레지스터(Register for operand B)
  • 두 레지스터의 값이 가산기(ADDER)에 입력되어 덧셈 연산을 수행한다
Page 24

레지스터의 데이터 저장 메커니즘

레지스터의 ENABLE 신호(OE)가 어서트되면 입력 데이터가 저장되고, 디어서트되면 기존 값이 유지된다.
상세 설명

레지스터의 저장 동작은 ENABLE 신호(OE, Output Enable)에 의해 제어된다:

  • OE가 디어서트(de-asserted)되면: 출력(Q)이 이전 값을 유지한다. 새로운 입력이 들어와도 무시된다.
  • OE가 어서트(asserted)되면: 출력이 입력 데이터(D)로 갱신된다. 내부 D 플립플롭이 값을 전파한다.

이 메커니즘을 통해 원하는 시점에만 데이터를 저장하고, 나머지 시간에는 기존 데이터를 안전하게 보존할 수 있다.

FSM 구조: 무어 머신 (Moore Machine)

무어 머신(Moore Machine)에서는 다음 상태가 현재 상태와 입력에 의해 결정되지만, 출력은 오직 현재 상태에 의해서만 결정된다.
상세 설명

무어 머신(Moore Machine)의 두 가지 핵심 규칙:

  1. 다음 상태(Next State) = f(현재 상태, 입력) — 조합 논리(CL)로 계산
  2. 출력(Output) = g(현재 상태) — 입력에 의존하지 않음

구조적으로 보면:

  • 첫 번째 조합 논리(CL): 현재 상태 + 입력 → 다음 상태 계산
  • 플립플롭(FF): 다음 상태를 저장하여 현재 상태로 만듦
  • 두 번째 조합 논리(CL): 현재 상태 → 출력 생성

무어 머신의 장점은 출력이 입력에 직접 연결되지 않아 글리치(glitch)가 적고 안정적이라는 것이다.

Page 26

FSM 구조: 밀리 머신 (Mealy Machine)

밀리 머신(Mealy Machine)에서는 출력이 현재 상태와 입력 모두에 의해 결정된다.
상세 설명

밀리 머신(Mealy Machine)의 두 가지 핵심 규칙:

  1. 다음 상태(Next State) = f(현재 상태, 입력) — 무어 머신과 동일
  2. 출력(Output) = g(현재 상태, 입력) — 입력도 출력에 영향

무어 머신과의 차이점은 출력을 결정하는 조합 논리에 입력이 직접 연결된다는 것이다. 이로 인해:

  • 장점: 같은 기능을 더 적은 상태로 구현할 수 있다 (상태 수가 줄어듦)
  • 단점: 입력 변화가 출력에 즉각 반영되므로 글리치(glitch)가 발생할 수 있다
시험 포인트: "Moore vs Mealy 차이는?" — 핵심은 출력 함수에 입력이 포함되느냐 여부이다. Moore: Output = g(State), Mealy: Output = g(State, Input).

모듈 (Module)

모듈(Module)은 Verilog에서 설계의 기본 단위이다. 보통 하나의 소스 파일에 하나의 모듈이 포함된다.
상세 설명

Module은 Verilog 설계의 기본 단위이다. 보통 하나의 소스 파일(source_code.v)에 하나의 모듈이 포함된다.

모듈은 두 부분으로 구성된다:

  • 선언부(Declaration): 모듈 내부에서 사용되는 변수들 (input, output, 신호, 함수 등)
  • 문장부(Statements): 모듈의 동작을 기술 (다른 인스턴스화된 모듈 포함 가능)
module moduleA();
  input B, C, D;          // 선언부

  assign result = B+C+D;  // 문장부
  moduleB(B);             // 하위 모듈 인스턴스화
  moduleC(C);
  moduleD(D);
endmodule

하나의 모듈 내부에 다른 모듈을 인스턴스화(instantiation)할 수 있다. 이를 통해 계층적(hierarchical) 설계가 가능하다.

토큰 (Tokens)

Verilog의 문장(statement)은 토큰(token)과 숫자로 구성된다. 토큰에는 키워드(keyword)와 식별자(identifier)가 있다.
상세 설명

키워드(Keywords = reserved words):

  • Verilog 명세에서 미리 정의된 이름 (예: module, input, output, wire, reg 등)
  • 사용자가 키워드를 변수 이름으로 사용할 수 없다

식별자(Identifiers):

  • 사용자가 정의하는 이름 — 대소문자 구분(case-sensitive)
  • 명명 규칙: 문자(letter) 또는 밑줄(_)로 시작해야 한다
  • 중간에 문자, 숫자, 밑줄, 달러 기호($)를 포함할 수 있다
module example(result, in1, in2);
  input in1, in2;
  output result;
  assign result = in1 + in2;
endmodule
Page 30

토큰 — 스타일 규칙

Verilog 코딩 시 대소문자 구분과 명명 규칙(naming convention)을 잘 지켜야 코드의 가독성과 유지보수성이 높아진다.
상세 설명

대소문자 규칙:

  • 소문자: 모든 신호(signal), 변수(variable), 포트(port) 이름에 사용
  • 대문자: 상수(constant)와 사용자 정의 타입(user-defined type) 이름에 사용

명명 규칙(Naming conventions):

  • 접미사 "_r": 레지스터된(registered) 값임을 표시
  • 접미사 "_w": 와이어(wired) 값임을 표시
  • 접미사 "_L" 또는 "_n": Active-Low 신호임을 표시

단, 규칙은 프로젝트마다 다를 수 있다. 중요한 것은 팀 내에서 일관성을 유지하는 것이다.

Verilog에서의 숫자 표현 (1)

Verilog는 4가지 논리 값(0, 1, z, x)을 지원하며, 강력한 비트별 연산(bitwise operations)을 제공한다.
상세 설명

4-value logic (4값 논리):

  • 0: 논리 0, 거짓(false) 조건
  • 1: 논리 1, 참(true) 조건
  • z: 고임피던스(high-impedance) — 외부로 신호를 전파하지 않음
    • 연결되지 않은(open) 상태를 나타냄
    • "?"로 표현 가능: 01?? = 01zz
    • inout 포트에서 자주 발생
  • x: 불확정(unknown), don't care
    • 초기값이 없을 때 발생
    • 중복 할당(duplicate allocation)으로 충돌 시 발생

비트별 연산자(Bitwise operators):

연산자동작
&AND
|OR
~NOT (물결/tilde)
^XOR (캐럿/caret)
~^ 또는 ^~XNOR
Page 32

Verilog에서의 숫자 표현 (2)

Verilog에서 숫자는 크기 지정(sized)과 크기 미지정(unsized) 두 가지 형식으로 표현하며, 밑줄(_)로 가독성을 높일 수 있다.
상세 설명

리터럴 형식(Literal formats):

  • 크기 지정(Sized numbers): <size>`<base><digits>
    • 예: 4'b1001 (4비트 이진수), 16'habcd (16비트 16진수)
  • 크기 미지정(Unsized numbers): `<base><digits>
    • 암묵적으로 32비트로 고정
    • 예: 2009, 'habc

진법(Base):

  • b 또는 B: 이진(binary)
  • o 또는 O: 8진(octal)
  • h 또는 H: 16진(hexadecimal)
  • d 또는 D: 10진(decimal)

가독성을 위한 구분자(separator): 긴 숫자의 자릿수를 확인하기 어려울 때 밑줄(_)을 사용한다.

16'b0101_1001_1110_0000  // 밑줄로 4비트씩 구분

조합 논리 구현 (Implementing Combinational Logic)

Verilog에서 조합 논리는 "assign" 또는 "always @(*)"로 구현하며, 반드시 블로킹 할당(=)만 사용해야 한다.
상세 설명

조합 논리 구현의 핵심 규칙:

  • "assign" 또는 "always @(*)" 사용
  • "=" 블로킹 할당(blocking assignment)만 사용! ("<="는 순차 논리용)
  • 모든 경우를 "else"까지 정의해야 한다 (그렇지 않으면 래치가 생성됨)

슬라이드의 MUX 기반 조합 논리 예시 (두 가지 스타일):

// 스타일 1: always 블록 사용
module example(
  input [3:0] N,
  input enable,
  output reg OUT);
  reg mux_w;
  always @(*) begin
    if (N[3]==1'b1)
      mux_w = N[0] & (N[1]^N[2]);
    else
      mux_w = N[0] | (N[1]^~N[2]);
    OUT = mux_w & enable;
  end
endmodule
// 스타일 2: assign 사용
module example(
  input [3:0] N,
  input enable,
  output OUT);
  wire mux_w;
  assign mux_w = N[3]? (N[0] & (N[1]^N[2]))
                      :(N[0] | (N[1]^~N[2]));
  assign OUT = enable & mux_w;
endmodule
주의: 조합 논리에서 모든 경우를 정의하지 않으면(else 누락) Verilog 합성 도구가 의도치 않은 래치(latch)를 생성한다. 이는 타이밍 문제의 주요 원인이 된다.
Page 34

순차 논리 구현 (Implementing Sequential Logic)

순차 논리는 "always @(posedge ...)"로 구현하며, 반드시 넌블로킹 할당(<=)만 사용해야 한다.
상세 설명

순차 논리 구현의 핵심 규칙:

  • "always @(posedge ...)" 사용 (또는 사양에 따라 negedge)
  • "<=" 넌블로킹 할당(non-blocking assignment)만 사용!

8비트 데이터 레지스터 예시:

always @(posedge clk) begin
  if (enable)
    data_out <= data_in;   // 넌블로킹 할당
end

블로킹(=)과 넌블로킹(<=)의 차이:

  • 블로킹(=): 문장이 순차적으로 실행. 조합 논리에 적합
  • 넌블로킹(<=): 모든 우변(RHS)을 먼저 평가한 후 동시에 좌변(LHS)에 할당. 순차 논리에 적합
핵심 규칙: 조합 논리 = always @(*) + 블로킹(=), 순차 논리 = always @(posedge clk) + 넌블로킹(<=). 이 규칙을 반드시 지켜야 한다.

모듈 간 연결: 사례 연구

서로 다른 모듈을 연결할 때는 인스턴스화(instantiation), 이름 연결(named association), 파라미터화(parameterization)를 사용한다.
상세 설명

모듈 간 연결의 핵심 원칙:

  • 모듈 인스턴스화: 하위 모듈을 상위 모듈 안에서 생성
  • 이름 연결(named association): .포트이름(와이어이름) 형식으로 명시적 연결
  • 비트 폭 파라미터화: 유연하고 모듈화된 연결을 위해 비트 폭을 파라미터로 설정
  • 비트 폭 일치: 연결된 와이어와 포트의 비트 폭이 일치해야 한다
module top();
  input i_data;
  output o_data;
  wire p2i_w, i2m_w, wpath_w, rpath_w;

  interface #(...) itf_0(
    .m2s0(i2m_w), .s2m0(p2i_w),
    .m2s1(o_data), .s2m1(i_data));

  process_unit #(...) pu_0(
    .p2i(p2i_w),
    .wmem(wpath_w), .rmem(rpath_w));

  memory #(...) mem_0(
    .wport(wpath_w), .rport(rpath_w),
    .wport2(i2m_w));
endmodule

이 예시는 interface, process_unit, memory 세 모듈이 wire로 상호 연결된 top 모듈을 보여준다.

데이터 타입: 변수 (Variables)

Verilog의 데이터 저장 요소 중 실제 설계에서는 주로 reg와 integer 타입을 사용하며, 나머지 타입은 합성 불가능(not synthesizable)하다.
상세 설명

변수(variable)는 데이터를 저장하는 요소이다:

  • 다양한 타입이 있지만, 실무에서는 reg와 integer를 주로 사용한다
  • 나머지 타입은 합성 불가능(not synthesizable)하다 — 시뮬레이션에서만 사용
  • 값을 할당할 때는 "always" / "initial" 블록을 사용한다 (절차적 할당, procedural assignment)
module example();
  wire a;
  reg b;
  always @* begin
    b = ~a;  // 절차적 할당
  end
endmodule
중요: Verilog의 변수 타입 "reg"는 물리적 플립플롭이나 레지스터와 아무 관련이 없다! 단지 always 블록 안에서 할당할 수 있는 변수 타입일 뿐이다. 이름 때문에 혼동하기 쉬우니 주의하자.

포트 타입 (Port Types)

Verilog 포트는 input, output, inout 세 가지 타입이 있으며, 각각 사용 규칙이 다르다.
상세 설명
  • input: 모듈이 소비하는 신호
    • 변수(reg)를 input으로 사용할 수 없다
  • output: 모듈이 생산하는 신호
    • 소유 모듈 자체가 소비할 수 없다
    • 변수(reg)를 output으로 사용할 수 있다
  • inout: 입력 또는 출력으로 사용 가능 — 양방향(tri-state) 핀용
    • 변수(reg)를 inout으로 사용할 수 없다
module example(G1, G2, G3, A, Y);
  input G1, G2, G3;       // = input wire G1, G2, G3
  input wire [2:0] A;
  output reg [7:0] Y;
  ...
endmodule

암묵적으로 선언된 포트는 wire 타입이다. 슬라이드에서는 "같은 기능인데 비트 폭만 다른 새 모듈을 매번 만들어야 하나?"라는 질문을 던지며, 다음 페이지의 파라미터 개념으로 연결한다.

파라미터 (Parameters)

파라미터(parameter)는 상수를 표현하는 데 사용되며, 비트 폭을 유연하게 변경할 수 있어 범용적인 모듈 설계가 가능하다.
상세 설명

parameter는 상수값을 나타내는 데 사용된다 — 주로 비트 폭, 인덱스 등에 활용된다.

문법: parameter <identifier> = value;

// 파라미터가 있는 모듈 정의
module example #(
  parameter WIDTH_IN = 4,
  parameter WIDTH_OUT = 10)
(
  input [WIDTH_IN-1:0] in,
  output [WIDTH_OUT-1:0] out
);
  ...
endmodule

// 상위 모듈에서 인스턴스화 시 파라미터 오버라이드
module top;
  input [4:0] in_top;
  output [17:0] out_top;
  example #(
    .WIDTH_IN(5),
    .WIDTH_OUT(18)
  ) example_uut(
    .in(in_top),
    .out(out_top)
  );
endmodule
상수이므로 코드에서 대문자로 정의하는 것이 권장된다 (예: WIDTH_IN, MAX_SIZE).

벡터 (Vectors): 사용법

벡터는 여러 비트를 가진 데이터를 나타내며, net(wire), 변수(reg), 상수에 사용할 수 있다.
상세 설명

벡터(vector)는 다중 비트 데이터를 표현하는 방법이다.

형식:

wire [MSB:LSB] identifier;
reg  [MSB:LSB] identifier;

MSB(최상위 비트)가 LSB(최하위 비트)보다 반드시 클 필요는 없다.

예시:

reg [1:16] Zbus;
Zbus = 16'b1000_0110_1100_1010;
Zbus[01] = 1   // 인덱스 1의 비트값
Zbus[16] = 0   // 인덱스 16의 비트값
Page 41

벡터 연산 (1): 선택, 연결, 반복

벡터에 대해 비트 선택(indexing), 연결(concatenation), 반복(replication) 연산을 수행할 수 있다.
상세 설명

선택(Selection / Indexing):

  • [<index>]: 비트 선택
  • [a:b]: 상수를 이용한 부분 선택
  • [a+:n] / [a-:n]: 변수를 이용한 부분 선택 — [(a+n-1):a] / [a:(a+1-n)]
reg [1:16] Zbus;
Zbus = 16'b1000_0110_1100_1011;
Zbus[1:2] = 10      // 정상
Zbus[2:1]            // ERROR! (선언과 방향이 다름)

연결(Concatenation): {A, B, C, ...}

  • {2'b00, 2'b11} = 4'b0011

반복(Replication): {<횟수>{DATA}}

  • {2{8'h43}} = 16'h4343
  • {16{1'b0}} = 16'h0000
  • {2{byte1}, 3{byte2}} = {byte1, byte1, byte2, byte2, byte2}
Page 42

벡터 연산 (2): 부울 축소 연산

축소 연산(reduction operation)은 벡터를 비트별 연산으로 단일 비트로 줄이는 연산이다.
상세 설명

부울 축소 연산은 벡터 자기 자신에 대해 비트별 연산을 수행하여 1비트 결과를 만들어낸다.

축소 연산자: &, |, ^, ^~, ~^

예시:

  • |DATA: DATA 비트 중 하나라도 1이면 결과가 1'b1
  • &DATA: DATA 비트 중 하나라도 0이면 결과가 1'b0, 모두 1이면 1'b1
  • ^DATA: DATA에서 1의 개수가 홀수이면 결과가 1'b1

예를 들어, 4비트 데이터 DATA = 4'b1010이면:

  • |DATA = 1|0|1|0 = 1
  • &DATA = 1&0&1&0 = 0
  • ^DATA = 1^0^1^0 = 0 (1이 2개, 짝수)

벡터 연산 (3): 산술 연산

Verilog는 다양한 산술 연산을 지원하며, 기본 연산은 합성 가능하지만 일부 복잡한 연산은 합성 불가능하다.
상세 설명

기본 산술 연산 (합성 가능):

연산자동작
+덧셈 (Addition)
-뺄셈 (Subtraction)
*곱셈 (Multiplication)

복잡한 산술 연산 (대부분 합성 불가능):

연산자동작
/나눗셈 (Division)
%나머지 (Modulus)
**거듭제곱 (Exponentiation)

시프트 연산:

연산자동작비고
<<논리 좌시프트선택적 합성 가능
>>논리 우시프트선택적 합성 가능
<<<산술 좌시프트선택적 합성 가능
>>>산술 우시프트선택적 합성 가능

FPGA에서 곱셈은 DSP 블록에 매핑될 수 있다.

벡터 연산 (4): 부호 있는 산술

Verilog-2001부터 signed 타입을 지원하며, 부호 있는(signed) 연산과 부호 없는(unsigned) 연산이 섞일 때 의도치 않은 타입캐스팅에 주의해야 한다.
상세 설명
  • Verilog-2001부터 net과 variable 모두 signed로 정의 가능
  • 예: reg signed A; wire signed B;
  • 음수값은 2의 보수(2's complement)로 표현된다

의도치 않은 타입캐스팅 주의!

  • signed 피연산자와 unsigned 피연산자가 만나면, signed가 암묵적으로 unsigned로 캐스팅된다
  • 이를 방지하려면 내장 함수 $signed() / $unsigned()를 사용해야 한다
// 잘못된 코드 (in_2가 unsigned라 in_1도 unsigned로 캐스팅됨)
assign out = in_1 * in_2;

// 올바른 코드 (in_2를 signed로 명시적 변환)
assign out = in_1 * $signed({1'b0, in_2});
주의: signed * unsigned 연산은 unsigned로 처리된다! 반드시 $signed()로 감싸야 올바른 결과를 얻는다.

배열 (Array)

Verilog-2001부터 다차원 배열이 지원되며, 메모리 구조나 룩업 테이블(LUT)을 정의하는 데 사용된다.
상세 설명
  • Verilog-1995까지는 다차원 데이터 타입이 지원되지 않았다
  • Verilog-2001부터 배열 지원 — 메모리 구조, 룩업 테이블(LUT) 정의에 활용

논리식: 참값 (Truth Values)

Verilog에서 참값(truth value)은 1비트 값이며, 다중 비트 값에서는 0이 아닌 모든 값이 참(TRUE)이다.
상세 설명

논리식 A && B, A || B 등에서 사용되는 참값:

  • 참값은 1비트 값이다
  • True: 1'b1
  • False: 1'b0, 또는 x나 z가 포함된 값

다중 비트 값의 경우:

  • 0이 아닌 모든 값이 TRUE로 간주된다
  • 따라서 모든 비트가 0인 경우만 FALSE이다
  • 예: (4'b0000)? → FALSE

Verilog에서 논리 연산자는 결과적으로 1비트 크기의 참값을 산출한다.

논리식: 조건 연산자와 케이스 동등 연산자

상세 설명

조건 연산자 (Conditional Operator, ?):

  • 형식: <논리식> ? <참 표현식> : <거짓 표현식>
  • C 언어의 삼항 연산자와 동일하다

케이스 동등 연산자 (Case equality, ===, !==):

  • 일반 동등 연산자(==)와 유사하지만, 모든 비트 위치를 비교한다
  • x와 z도 정확히 비교한다 (== 연산자는 x/z가 있으면 결과가 x)
  • 합성 불가능(NOT synthesizable)하지만, 시뮬레이션에서 x나 z의 존재를 검사할 때 유용하다

함수 (Function)

상세 설명

태스크 (Task)

Task는 Function보다 범용적이며 타이밍 키워드를 포함할 수 있지만, 대부분 합성 불가능하여 시뮬레이션에 주로 사용된다.
상세 설명

Task의 특성:

  • Function보다 범용적: 타이밍 키워드(@, posedge 등) 포함 가능
  • 문장(statement)의 위치에서 사용 가능
  • 모듈 내부와 외부 모두에서 정의 가능

두 가지 종류:

  • Static task: 멤버 변수가 여러 호출 간에 공유됨
  • Automatic task: 멤버 변수가 여러 호출 간에 공유되지 않음
// Static task: i값이 공유됨        // Automatic task: i값이 독립적
task display();                    task automatic display();
  integer i = 0;                     integer i = 0;
  i = i + 1;                         i = i + 1;
  $display("i=%0d", i);              $display("i=%0d", i);
endtask                            endtask
// 결과: i=1, i=2, i=3             // 결과: i=1, i=1, i=1
핵심: Function = 합성 가능, 조합 논리용. Task = 대부분 합성 불가능, 시뮬레이션용.

성능? 트레이드오프의 문제 (Trade-Off)

시스템 성능 최적화는 결국 트레이드오프(trade-off) 지점을 찾는 것이다. 에너지, 시간, 비용, 보안 등 여러 요소가 상충한다.

성능은 에너지, 시간, 신뢰성, 비용, 보안 등의 함수이다. 어떤 하나를 개선하면 다른 것이 희생되는 경우가 많다.

성능 지표 (Performance Metrics)

높은 성능이란 높은 처리량(Throughput) 또는 낮은 지연 시간(Latency)을 의미한다. 이 둘은 서로 다른 관점의 지표이다.
상세 설명

처리량(Throughput):

  • CPU에서는 IPC(Instructions Per Cycle)로 반영

  • 작업의 시작부터 완료까지 걸린 시간
  • 보통 프로그램/사용자 관점에서 측정
  • 시간(time) 측정이다

Throughput == 1 / Latency 인가?

처리량은 지연 시간의 역수가 아닐 수 있다. 파이프라이닝(pipelining)을 사용하면 개별 작업의 지연은 늘어나도 전체 처리량은 증가한다.
상세 설명

하드웨어 파이프라이닝의 예를 통해 이해해보자:

파이프라인 없이:

  • 하나의 로직 블록 (40ns)
  • Task A: 40ns, Task B: 40ns, Task C: 40ns
  • 총 시간: 120ns

3단계 파이프라인 사용 (clk=20ns):

  • Stage 1: 10ns, Stage 2: 20ns, Stage 3: 10ns
  • 각 Task의 개별 지연: 60ns (40ns보다 오히려 증가!)
  • 총 시간: 100ns (120ns에서 감소)

파이프라이닝은 여러 작업을 겹쳐서(concurrently) 실행하므로, 개별 작업의 지연은 증가하지만 전체 작업의 처리량(throughput)은 증가한다.

핵심: 높은 처리량이 반드시 낮은 지연 시간을 의미하지 않는다. 동시성(concurrency)을 고려하면 두 지표는 독립적일 수 있다. 다만, 전체 작업들의 총 지연은 줄어든다.
Page 56

처리량 지표: 비율 지표 (Rate Metrics)

대표적인 비율 지표로 IPC, MIPS, GHz가 있으며, 이들을 조합하여 벽시계 시간(wall clock time)을 계산할 수 있다.
상세 설명

대표적인 비율 지표:

  • IPC: 사이클당 명령어 수 (Instructions Per Cycle)
  • MIPS: 초당 백만 명령어 (Million Instructions Per Second) — MIPS ISA와 혼동 주의!
  • GHz: 초당 십억 사이클 (Billion Cycles Per Second)

비율에서 시간으로 변환:

\[ \text{Wall clock time} = \frac{1}{\text{GHz}} \times \frac{1}{\text{IPC}} \times \text{명령어 수} \]

차원 분석: 시간/사이클 × 사이클/명령어 × 명령어 수 = 시간

각 요소의 결정 인자:

  • 명령어 수: ISA와 컴파일러에 의해 결정
  • 1/GHz: 주파수와 공정 기술에 의해 결정
  • 1/IPC: 마이크로아키텍처에 의해 결정

MIPS와 IPC는 평균(average) 비율 값이다. 또한, 이 공식은 단일 코어 추정에 적합하다.

주의: 마이크로아키텍처 개선이 IPC를 감소시킬 수도 있다! 멀티프로세서 워크로드에서 IPC는 오해를 유발할 수 있다 (Alameldeen et al., "IPC CONSIDERED HARMFUL FOR MULTIPROCESSOR WORKLOADS," IEEE Micro, 2006).

Throughput vs. Bandwidth

Throughput과 Bandwidth는 단위가 거의 같지만 서로 다른 지표이다. Bandwidth는 이론적 최대 능력, Throughput은 실제 처리량이다.
상세 설명

대역폭(Bandwidth):

  • 단위 시간 내에 전송 가능한 최대 데이터 양
  • 처리의 이상적인 능력을 나타냄 (보통 이론적 전송량)

처리량(Throughput):

  • 단위 시간 내에 실제로 처리된 데이터/계산의 양
  • 주어진 상황(예: 애플리케이션, 설정)에서의 실제 성능을 나타냄

비유: 고속도로의 차선 수가 bandwidth(동시에 통과할 수 있는 최대 차량 수)이고, 실제 교통 상황에서 통과하는 차량 수가 throughput이다.

Throughput ≤ Bandwidth. 실제 처리량은 이론적 대역폭을 초과할 수 없다.

FLOPS (Floating-Point Operations)

FLOPS는 프로그램 실행 중 수행된 부동소수점 연산의 횟수이며, 과학 계산 분야에서 널리 사용되는 성능 지표이다.
상세 설명
  • FLOPS: 프로그램 실행 동안의 부동소수점 연산(floating-point operation) 횟수
  • 과학 계산(scientific computing) 연구자들이 주로 사용한다
    • 과학 애플리케이션은 대부분 실수(real number) 연산이 필요
    • 부동소수점 연산은 일반적으로 알고리즘 복잡도가 높다
  • 과학 계산 워크로드에 좋은 지표이다

주의: FLOPS는 비율(rate)이 아니다. 단순히 부동소수점 연산의 개수를 세는 것이다. 모든 알고리즘은 명령어 수가 다르므로, 단독으로는 공정한 비교 지표가 되기 어렵다.

FLOPS와 FLOPS/s를 구분하자. FLOPS 자체는 연산 횟수이고, FLOPS/s (= FLOP/s)가 초당 연산 비율이다.

하드웨어 성능 시각화: 루프라인 모델 (Roofline Model)

루프라인 모델은 알고리즘의 산술 강도(Arithmetic Intensity)와 하드웨어의 대역폭(Bandwidth)을 이용하여 성능의 상한을 시각화하는 모델이다.
상세 설명

FLOPS를 떠올리면:

\[ \text{FLOPS per second} = AI \times BW \]
  • AI (Arithmetic Intensity, 산술 강도): 데이터 1바이트당 수행되는 부동소수점 연산 수 — 알고리즘에 의존적 (데이터 재사용률)
  • BW (Bandwidth, 대역폭): 최대 처리 가능한 데이터 양 — 하드웨어에 의존적

양변에 로그를 취하면:

\[ \log(\text{FLOPS/s}) = \log(AI) + \log(BW) \] \[ \Rightarrow y = x + b \]

이를 그래프로 그리면 지붕(roofline) 모양이 나온다:

  • 왼쪽 영역 (기울기 1 직선): 메모리 제한(memory-bounded) 영역. 데이터 전송에 대부분의 시간을 소비
  • 오른쪽 영역 (수평선): 계산 제한(compute-bounded) 영역. 하드웨어 연산 능력에 의해 제한

프로그램 X가 왼쪽에 위치하면 메모리 바운드, 프로그램 Y가 오른쪽에 위치하면 컴퓨트 바운드이다.

루프라인 모델의 실용적 가치: 내 프로그램이 메모리 병목인지 계산 병목인지 한눈에 판단할 수 있다. 메모리 바운드라면 데이터 접근 패턴을 최적화하고, 컴퓨트 바운드라면 연산 효율을 높여야 한다.
Page 60

다양한 하드웨어의 루프라인 예시

TPU, K80(GPU), HSW(CPU) 등 서로 다른 하드웨어가 각기 다른 루프라인 형태를 보여준다.
상세 설명

슬라이드에서는 세 가지 하드웨어의 루프라인을 비교한다:

  • TPU: Google의 Tensor Processing Unit — 딥러닝 특화 프로세서
  • K80: NVIDIA의 GPU (Kepler 아키텍처)
  • HSW: Intel의 Haswell CPU 아키텍처

각 하드웨어마다 컴퓨트 성능(수평선의 높이)과 메모리 대역폭(기울기 직선의 위치)이 다르다. TPU는 딥러닝 워크로드에 최적화되어 높은 연산 처리량을 보여주며, CPU는 범용적이지만 특화 성능은 상대적으로 낮다.

상대 성능 (Relative Performance)

시스템의 성능이 "좋다" 또는 "나쁘다"를 판단하려면 기준선(baseline) 시스템과 비교하는 상대 성능이 필요하다.
상세 설명
  • 시스템 A는 높은 처리량을 가지고, 시스템 B는 낮은 지연 시간을 가진다고 하자
  • 그것이 "높다" 또는 "낮다"는 것을 어떻게 알 수 있는가?
  • → 상대 성능(relative performance)을 사용한다

상대 성능의 핵심:

  • 성능의 품질을 보여주는 방법
  • 기준선(baseline) 시스템이 필요하다 — 이를 통해 시스템이 좋은지 나쁜지 판단 가능
Page 62

상대 성능: 백분율 변화 (Percentage Change)

백분율 변화에서 분모가 기준값(reference)이라는 점이 핵심이다. "X보다 Y가 50% 빠르다"의 정확한 의미를 이해해야 한다.
상세 설명

예제: 시스템 X의 시간이 \( t_X = 1.0s \)이고, X가 Y보다 50% 느리다. \( t_Y \)는?

  • Case 1: \( \frac{t_X - t_Y}{t_X} = 50\% \) → \( t_Y / t_X = 0.5 \) → \( t_Y = 0.5s \)
  • Case 2: \( \frac{t_X - t_Y}{t_Y} = 50\% \) → \( t_Y / t_X = 1/1.5 \) → \( t_Y = 0.666s \)

상대 변화(relative change)의 정의에 집중하자:

  • 분모는 기준값(reference value) 또는 이전 값(old value)이다
  • 즉, "v는 vref에 대한 변화"라는 의미 — Case 1이 표준적 정의이다

이 강의에서는 Case 1의 정의를 따른다.

주의: "50% 느리다"에서 분모가 무엇인지에 따라 답이 완전히 달라진다! 항상 기준값이 무엇인지 명확히 해야 한다.

상대 성능: 정규화 성능 (Normalized Performance)

두 성능의 비율을 직접 취하는 것을 정규화 성능 또는 스피드업(Speedup)이라 하며, 백분율 변화와 관계를 가진다.
상세 설명

정규화 성능(Normalized Performance):

  • 두 성능의 비율을 직접 계산
  • 때때로 스피드업(speedup)이라고도 한다

"X가 Y보다 n배 빠르다"의 의미:

\[ n = \frac{\text{Performance}_X}{\text{Performance}_Y} = \frac{\text{Throughput}_X}{\text{Throughput}_Y} = \frac{\text{Time}_Y}{\text{Time}_X} \]

백분율 변화와의 관계:

"X가 Y보다 m% 빠르다"라면:

\[ 1 + m\% = \frac{\text{Performance}_X}{\text{Performance}_Y} \]

성능의 평균 (Means of Performance)

시스템의 성능은 프로그램/워크로드에 따라 달라지므로, 평균적/일반적 경향을 파악하기 위해 적절한 평균(mean)을 사용해야 한다.
상세 설명
  • 시스템의 성능은 다양한 요인(특히 프로그램/워크로드)에 따라 달라진다
  • 서로 다른 프로그램은 서로 다른 성능을 보인다
  • 평균적/일반적 경향을 알고 싶다 — 평균(mean)이 핵심 방법이다

하지만 아무 평균이나 사용하면 안 된다! 데이터의 성격에 따라 올바른 평균을 선택해야 한다:

  • 절대값(absolute numbers)에 대해서는 산술 평균(Arithmetic Mean)
  • 정규화된 값(normalized numbers)에 대해서는 기하 평균(Geometric Mean)
  • 비율(rate)에 대해서는 조화 평균(Harmonic Mean)

산술 평균 (Arithmetic Mean)

산술 평균은 절대적인 수치(실행 시간, 처리량, 지연 시간 등)의 평균을 구할 때 사용하며, 가중치를 적용할 수도 있다.
상세 설명

단순 산술 평균(Naive Arithmetic Mean):

\[ \overline{\text{performance}} = \sum_{i=0}^{n-1} \text{performance}_{AM_i} \]

가중 산술 평균(Weighted Arithmetic Mean):

  • 애플리케이션/워크로드마다 실행 길이가 다르다
  • 각 애플리케이션에 대한 가중치(weight)를 고려해야 한다
\[ \overline{\text{performance}} = \sum_{i=0}^{n-1} \text{weight}_i \times \text{performance}_{AM_i}, \quad \sum_{i=0}^{n-1} \text{weight}_i = 1 \]

기하 평균 (Geometric Mean) (1)

기하 평균은 정규화된 수치(스피드업, 정규화 에너지 등)의 평균에 사용하며, 산술 평균을 정규화 값에 사용하면 잘못된 결과가 나온다.
상세 설명
\[ \text{normalized metric} = \sqrt[n]{\prod_{i=0}^{n-1} \text{normalized metric}_{AM_i}} \]

중요: 정규화된 수치에 산술 평균을 사용하면 안 된다!

슬라이드의 실제 데이터 예시 (3개 시스템, 8개 애플리케이션):

방법PerfX/PerfBasePerfY/PerfBasePerfY/PerfX
기하 평균a=0.2388b=1.1313b/a = 4.7360
산술 평균c=0.2748d=1.1359d/c = 4.13

기하 평균의 비율(b/a = 4.7360)은 직접 계산한 PerfY/PerfX의 기하 평균과 일치한다. 반면, 산술 평균의 비율(d/c = 4.13)은 직접 계산한 값(5.8677)과 큰 차이가 있다.

핵심 경고: 정규화된 성능(스피드업 등)에 산술 평균을 사용하면, 기준 시스템에 따라 결과가 달라지는 일관성 없는(inconsistent) 결과를 얻는다! 반드시 기하 평균을 사용해야 한다.

기하 평균 (Geometric Mean) (2) — 수학적 증명

기하 평균은 비율의 비율을 취해도 일관된 결과를 보장하지만, 산술 평균은 그렇지 않다는 것을 수학적으로 보여준다.
상세 설명

기하 평균(G.M.)의 일관성 증명:

\[ \frac{G.M.\left(\frac{Perf_Y}{Perf_{base}}\right)}{G.M.\left(\frac{Perf_X}{Perf_{base}}\right)} = \frac{\sqrt[8]{Perf_{Y,A1} \cdots Perf_{Y,A8}}}{\sqrt[8]{Perf_{X,A1} \cdots Perf_{X,A8}}} \times \frac{\sqrt[8]{Perf_{base,A1} \cdots Perf_{base,A8}}}{\sqrt[8]{Perf_{base,A1} \cdots Perf_{base,A8}}} \]

분모의 baseline 부분이 약분되어:

\[ = \sqrt[8]{\frac{Perf_{Y,A1} \cdots Perf_{Y,A8}}{Perf_{X,A1} \cdots Perf_{X,A8}}} = G.M.\left(\frac{Perf_Y}{Perf_X}\right) \]

즉, 기하 평균은 baseline이 무엇이든 비율의 비율이 일관적이다.

산술 평균(A.M.)의 경우:

\[ \frac{A.M.\left(\frac{Perf_Y}{Perf_{base}}\right)}{A.M.\left(\frac{Perf_X}{Perf_{base}}\right)} = \frac{\frac{Perf_{Y,A1}/Perf_{base,A1} + \cdots}{8}}{\frac{Perf_{X,A1}/Perf_{base,A1} + \cdots}{8}} \neq A.M.\left(\frac{Perf_Y}{Perf_X}\right) \]

산술 평균에서는 baseline이 약분되지 않아 일관성이 보장되지 않는다.

조화 평균 (Harmonic Mean)

조화 평균은 비율(rate)의 평균을 구할 때 사용하며, IPC, MIPS 등 처리량 지표의 평균에 적합하다.
상세 설명

조화 평균은 비율의 평균에 사용된다:

  • 대표적 예: 처리량(throughput) — IPC, MIPS 등

IPC = 1/CPI를 떠올려보자 (CPI는 지연 시간 지표):

"A.M.(CPI)의 역수"는 무엇인가?

\[ A.M.(CPI)^{-1} = \frac{n}{CPI_1 + CPI_2 + \cdots + CPI_n} = \frac{n}{\frac{1}{IPC_1} + \frac{1}{IPC_2} + \cdots + \frac{1}{IPC_n}} = H.M.(IPC) \]

즉, 지연 시간(CPI)의 산술 평균의 역수는 비율(IPC)의 조화 평균과 같다! 이것이 비율에 조화 평균을 사용해야 하는 이론적 근거이다.

평균 선택 요약:
• 절대값 (실행시간, 에너지) → 산술 평균 (Arithmetic Mean)
• 정규화된 값 (스피드업) → 기하 평균 (Geometric Mean)
• 비율 (IPC, MIPS) → 조화 평균 (Harmonic Mean)
