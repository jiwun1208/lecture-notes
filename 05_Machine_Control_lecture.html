프로세서 상태 — Condition Codes

x86-64 프로세서의 상태 정보에는 16개의 레지스터 외에 Condition Codes(조건 코드)라는 1비트 플래그들이 있습니다:

상태저장 위치용도
임시 데이터%rax 등 범용 레지스터연산 중간 결과
런타임 스택 위치%rsp스택의 현재 최상단
현재 명령어 위치%rip (Program Counter)다음 실행할 명령어 주소
최근 연산 결과 상태CF, ZF, SF, OF조건 분기의 기준
Condition Codes는 별도의 레지스터가 아닌 1비트 플래그입니다. 산술/논리 연산이 실행될 때마다 자동으로 갱신되며, 이 값을 읽어서 조건 분기를 결정합니다.
Page 05

Condition Codes — 산술 연산에 의한 설정

4개의 단일 비트 레지스터:

플래그이름의미
CFCarry Flagunsigned 연산에서 올림/빌림 발생
ZFZero Flag결과가 0
SFSign Flag결과가 음수 (최상위 비트가 1)
OFOverflow Flagsigned 연산에서 오버플로우 발생

예시: addq Src, Dest (t = a + b)를 실행하면:

CF: 최상위 비트에서 carry out 발생 시 (unsigned overflow)
ZF: t == 0이면 set
SF: t < 0이면 set (signed로 해석 시)
OF: signed overflow — (a>0 && b>0 && t<0) || (a<0 && b<0 && t>=0)
leaq는 Condition Codes를 설정하지 않습니다! leaq는 주소 계산용이지 산술 연산이 아니기 때문입니다.
Page 06

cmpq — 비교 명령어

cmpq Src2, Src1 — a - b를 계산하지만 결과를 저장하지 않고 Condition Codes만 설정합니다.

cmpq b, a    # a - b를 계산, 결과는 버림, 플래그만 설정
플래그설정 조건의미
CFMSB에서 carry outunsigned 비교에 사용
ZFa == b같은 값
SF(a - b) < 0signed로 a가 작음
OFsigned overflow(a>0 && b<0 && (a-b)<0) || (a<0 && b>0 && (a-b)>0)
cmpq는 subq와 동일한 연산을 하지만 결과를 어디에도 저장하지 않습니다. 오직 플래그 설정만이 목적입니다. if (a > b)를 구현할 때 핵심적으로 사용됩니다.
Page 07

testq — 테스트 명령어

testq Src2, Src1 — a & b를 계산하지만 결과를 저장하지 않고 Condition Codes만 설정합니다.

testq b, a    # a & b를 계산, 결과는 버림, 플래그만 설정
플래그설정 조건
ZFa & b == 0일 때
SFa & b < 0일 때

testq의 대표적 사용법:

testq %rax, %rax    # rax & rax = rax의 값 자체를 테스트
                     # ZF: rax == 0이면 set
                     # SF: rax < 0이면 set
testq %rax, %rax는 가장 흔한 패턴입니다. 같은 레지스터를 두 번 넣으면 AND 결과가 자기 자신이므로, 해당 값이 0인지/음수인지를 간편하게 검사할 수 있습니다.
Page 08

SetX 명령어 — Condition Codes 읽기

SetX 명령어는 Condition Codes의 조합에 따라 목적지의 하위 1바이트를 0 또는 1로 설정합니다.

명령어조건의미
seteZFEqual / Zero
setne~ZFNot Equal / Not Zero
setsSFNegative
setns~SFNonnegative
setg~(SF^OF) & ~ZFGreater (Signed)
setge~(SF^OF)Greater or Equal (Signed)
setlSF^OFLess (Signed)
setle(SF^OF) | ZFLess or Equal (Signed)
seta~CF & ~ZFAbove (Unsigned)
setbCFBelow (Unsigned)
주의: SetX는 1바이트만 설정하고 나머지 7바이트는 건드리지 않습니다!
Page 09

x86-64 하위 바이트 레지스터

SetX 명령어의 대상은 레지스터의 하위 1바이트입니다:

64비트     하위 1바이트     64비트     하위 1바이트
%rax       %al             %r8        %r8b
%rbx       %bl             %r9        %r9b
%rcx       %cl             %r10       %r10b
%rdx       %dl             %r11       %r11b
%rsi       %sil            %r12       %r12b
%rdi       %dil            %r13       %r13b
%rsp       %spl            %r14       %r14b
%rbp       %bpl            %r15       %r15b
x86-64에서는 하위 1바이트(%al), 2바이트(%ax), 4바이트(%eax), 8바이트(%rax)를 모두 개별적으로 접근할 수 있습니다. SetX는 이 중 1바이트 레지스터를 사용합니다.
Page 10

SetX 사용 예제 — gt 함수

int gt(long x, long y) {
    return x > y;
}

컴파일 결과 (gcc -Og):

gt:
    cmpq   %rsi, %rdi    # x - y 비교 (x:y)
    setg   %al            # x > y이면 %al = 1, 아니면 0
    movzbl %al, %eax      # %al을 %eax로 zero-extend
    ret                   # %eax(= %rax 하위 4바이트)를 반환
레지스터용도
%rdi첫 번째 인자 x
%rsi두 번째 인자 y
%rax반환값 (0 또는 1)
movzbl %al, %eax: "Move Zero-extending Byte to Long". %al(1바이트)을 %eax(4바이트)로 확장하면서 상위 비트를 0으로 채웁니다. 32비트 연산이 자동으로 상위 32비트도 0으로 만들기 때문에 결과적으로 %rax 전체가 깨끗해집니다.

로드맵 — 조건부 분기

— Conditional Branches —

Condition Codes를 설정하는 법을 배웠으니, 이제 이를 이용해 프로그램의 실행 흐름을 바꾸는 방법을 배웁니다. if-else 문이 어셈블리에서 어떻게 구현되는지 살펴봅니다.

Page 12

jX 명령어 — 조건부 점프

jX 명령어는 Condition Codes에 따라 다른 코드 위치로 점프합니다:

명령어조건의미
jmp1 (항상)Unconditional jump
jeZFEqual / Zero
jne~ZFNot Equal / Not Zero
jsSFNegative
jns~SFNonnegative
jg~(SF^OF) & ~ZFGreater (Signed)
jge~(SF^OF)Greater or Equal (Signed)
jlSF^OFLess (Signed)
jle(SF^OF) | ZFLess or Equal (Signed)
ja~CF & ~ZFAbove (Unsigned)
jbCFBelow (Unsigned)
SetX와 jX의 조건이 완전히 동일합니다! SetX는 결과를 바이트에 저장하고, jX는 점프 여부를 결정하는 차이만 있습니다.
Page 13

조건 분기 예제 — absdiff

long absdiff(long x, long y) {
    long result;
    if (x > y)
        result = x - y;
    else
        result = y - x;
    return result;
}

컴파일 결과 (gcc -Og -S -fno-if-conversion):

absdiff:
    cmpq    %rsi, %rdi    # x:y 비교
    jle     .L4           # x <= y이면 .L4로 점프
    movq    %rdi, %rax    # result = x
    subq    %rsi, %rax    # result = x - y
    ret
.L4:                      # x <= y인 경우
    movq    %rsi, %rax    # result = y
    subq    %rdi, %rax    # result = y - x
    ret
if의 조건(x > y)의 반대 조건(x ≤ y)으로 점프합니다. 조건이 거짓일 때 else 블록으로 뛰는 것이 일반적인 패턴입니다.
Page 14

Goto 코드로 표현하기

C의 goto 문을 사용하면 어셈블리의 동작을 정확히 흉내낼 수 있습니다:

// 원본 C 코드                    // Goto 버전
long absdiff(long x, long y)     long absdiff_j(long x, long y)
{                                 {
    long result;                      long result;
    if (x > y)                        int ntest = x <= y;
        result = x-y;                 if (ntest) goto Else;
    else                              result = x-y;
        result = y-x;                 goto Done;
    return result;                Else:
}                                     result = y-x;
                                  Done:
                                      return result;
                                  }
Goto 변환의 핵심 패턴:
1. 조건의 부정(ntest)을 계산
2. ntest가 참이면 Else 블록으로 goto
3. Then 블록 실행 후 Done으로 goto
4. Else 블록 실행
5. Done에서 합류
Page 15

일반적인 조건문 변환 패턴

C의 삼항 연산자 또는 if-else는 다음과 같은 일반적 패턴으로 어셈블리에 변환됩니다:

// C 코드                         // Goto 버전
val = Test ? Then_Expr              ntest = !Test;
           : Else_Expr;             if (ntest) goto Else;
                                    val = Then_Expr;
                                    goto Done;
                                  Else:
                                    val = Else_Expr;
                                  Done:
                                    ...

이 패턴을 분기(Branch) 기반 변환이라 합니다:

Then 블록과 Else 블록을 별도 코드 영역으로 생성
조건에 따라 하나만 실행
두 블록 모두 Done에서 합류

Conditional Move — 분기 없는 조건부 연산

Conditional Move (cmovX): 분기(jmp) 없이 조건부로 데이터를 이동하는 명령어입니다.

// C 코드                          // Goto 버전 (Conditional Move)
val = Test ? Then_Expr              result = Then_Expr;
           : Else_Expr;             eval = Else_Expr;
                                    nt = !Test;
                                    if (nt) result = eval;
                                    return result;

핵심 차이점:

분기 기반: Then 또는 Else 중 하나만 실행
Conditional Move: 둘 다 계산한 후, 조건에 따라 결과를 선택

왜 Conditional Move가 좋은가?

현대 CPU는 파이프라인(Pipeline)으로 명령어를 처리합니다. 분기(Branch)가 발생하면 파이프라인을 비우고 다시 채워야 하므로 큰 성능 손실이 생깁니다. Conditional Move는 제어 흐름을 바꾸지 않으므로 파이프라인을 방해하지 않습니다.
Page 17

Conditional Move 예제 — absdiff

long absdiff(long x, long y) {
    long result;
    if (x > y)
        result = x-y;
    else
        result = y-x;
    return result;
}

Conditional Move를 사용한 어셈블리:

absdiff:
    movq    %rdi, %rax    # rax = x
    subq    %rsi, %rax    # rax = x - y  (result 후보 1)
    movq    %rsi, %rdx
    subq    %rdi, %rdx    # rdx = y - x  (result 후보 2)
    cmpq    %rsi, %rdi    # x:y 비교
    cmovle  %rdx, %rax    # x <= y이면 rax = rdx (y-x 선택)
    ret
x-y와 y-x를 모두 계산한 후, cmpq로 비교하고 cmovle로 올바른 결과를 선택합니다. 점프가 전혀 없습니다!
Page 18

Conditional Move를 쓰면 안 되는 경우

Conditional Move는 양쪽을 모두 계산하므로, 다음 경우에는 사용할 수 없습니다:

경우예시이유
비용이 큰 계산val = Test(x) ? Hard1(x) : Hard2(x);양쪽 모두 계산하면 낭비가 큼
위험한 계산val = p ? *p : 0;p가 NULL이면 *p에서 segfault 발생!
부작용이 있는 계산val = x > 0 ? x*=7 : x+=3;양쪽 모두 실행하면 x가 두 번 수정됨
GCC의 판단: 컴파일러가 안전하다고 판단한 경우에만 Conditional Move를 사용합니다. -fno-if-conversion 옵션으로 강제로 끌 수 있습니다.

로드맵 — 루프

— Loops —

C의 do-while, while, for 루프가 어셈블리에서 어떤 패턴으로 변환되는지를 배웁니다. 모든 루프는 궁극적으로 조건부 점프(conditional jump)로 구현됩니다.

Page 20

Do-While 루프 예제 — popcount

popcount: 인자 x에서 1인 비트의 개수를 세는 함수입니다.

// C 코드                               // Goto 버전
long pcount_do(unsigned long x) {       long pcount_goto(unsigned long x) {
    long result = 0;                        long result = 0;
    do {                                  loop:
        result += x & 0x1;                  result += x & 0x1;
        x >>= 1;                            x >>= 1;
    } while (x);                            if (x) goto loop;
    return result;                          return result;
}                                        }
Do-While의 핵심: 루프 본문을 최소 1번 실행한 후 조건을 검사합니다. goto 버전에서는 loop 레이블 뒤에 본문이 먼저, 그 다음 조건 검사(if → goto)가 옵니다.
Page 21

Do-While 루프 컴파일 결과

// Goto 버전                           // 어셈블리
long pcount_goto(unsigned long x) {
    long result = 0;                    movl   $0, %eax     # result = 0
  loop:                                .L2:                  # loop:
    result += x & 0x1;                  movq   %rdi, %rdx
                                        andl   $1, %edx     #   t = x & 0x1
                                        addq   %rdx, %rax   #   result += t
    x >>= 1;                            shrq   %rdi         #   x >>= 1
    if (x) goto loop;                   jne    .L2          #   if (x) goto loop
    return result;                      ret
}
레지스터용도
%rdi인자 x (루프마다 오른쪽 시프트)
%raxresult (누적 카운터)
shrq %rdi 후 jne: shrq가 ZF를 설정하므로, x가 0이 아니면 루프 계속. 별도의 cmpq/testq 없이 직접 플래그를 활용합니다!
Page 22

Do-While 일반 변환 패턴

// C 코드               // Goto 버전
do                      loop:
    Body                    Body
while (Test);               if (Test)
                               goto loop

가장 단순한 루프 변환 패턴입니다:

1. loop 레이블 설정
2. Body 실행
3. Test 검사 → 참이면 loop로 goto
4. 거짓이면 루프를 빠져나옴

while과 for 루프도 궁극적으로 이 do-while 패턴으로 환원됩니다.

Page 23

While 루프 변환 #1 — Jump-to-Middle

Jump-to-Middle 방식 (-Og 최적화에서 사용):

// While 버전              // Goto 버전
while (Test)                  goto test;
    Body                   loop:
                               Body
                           test:
                               if (Test)
                                  goto loop;
                           done:
핵심 아이디어: 루프 시작 시 먼저 test로 점프합니다. 그래서 "Jump-to-Middle"이라는 이름입니다.

do-while과의 차이:

do-while: 무조건 본문 1번 실행 → 조건 검사
while (Jump-to-Middle): 먼저 조건 검사 위치로 점프 → 조건이 참이면 본문 실행 → 반복
Page 24

While 루프 예제 — Jump-to-Middle

// C 코드                                // Jump-to-Middle 버전
long pcount_while(unsigned long x) {     long pcount_goto_jtm(unsigned long x) {
    long result = 0;                         long result = 0;
    while (x) {                              goto test;
        result += x & 0x1;               loop:
        x >>= 1;                             result += x & 0x1;
    }                                        x >>= 1;
    return result;                        test:
}                                            if (x) goto loop;
                                             return result;
                                         }
do-while 버전과 비교: do-while은 바로 loop에서 시작하지만, 이 버전은 goto test로 먼저 x가 0인지 확인합니다. x가 처음부터 0이면 루프를 한 번도 실행하지 않습니다.

While 루프 변환 #2 — Do-While 변환

Do-While 변환 방식 (-O1 이상에서 사용):

// While 버전              // Do-While 변환              // Goto 버전
while (Test)               if (!Test)                     if (!Test)
    Body                       goto done;                     goto done;
                            do                            loop:
                                Body                          Body
                            while (Test);                     if (Test)
                            done:                                goto loop;
                                                          done:
핵심 아이디어:
1. 루프 진입 전 조건을 먼저 검사 (Guard Test)
2. 거짓이면 바로 done으로 점프
3. 참이면 do-while 루프 실행

Jump-to-Middle보다 약간 더 효율적입니다. goto가 루프 내부에 없기 때문에 분기 예측이 더 쉽습니다.

    long result = 0;                         long result = 0;

for 루프의 4가지 구성 요소:

for (Init; Test; Update)
    Body
구성 요소역할예시

#define WSIZE 8*sizeof(int)

    size_t i;
    long result = 0;
    for (i = 0; i < WSIZE; i++) {
        unsigned bit = (x >> i) & 0x1;
        result += bit;
    }
    return result;
}

For 루프 → While 루프 변환

컴파일러는 for 루프를 먼저 while 루프로 변환한 뒤, 이전에 배운 while 변환 기법을 적용합니다:

// For 버전                    // While 버전

    Body                       while (Test) {

변환 규칙:

2. Test를 while 조건으로

For 루프의 Do-While 변환

최종적으로 for 루프는 Do-While + Guard 형태의 goto 코드로 변환됩니다:

long pcount_for_goto_dw(unsigned long x) {
    size_t i;
    long result = 0;
    i = 0;                          // Init
    if (!(i < WSIZE))               // Guard: !Test → done
        goto done;
  loop:
    {
        unsigned bit =
            (x >> i) & 0x1;        // Body
        result += bit;
    }
    i++;                            // Update
    if (i < WSIZE)                  // Test → loop
        goto loop;
  done:
    return result;
}
초기 테스트 최적화: 여기서 Init이 i = 0이고 Test가 i < WSIZE(= 32)이므로, 초기 Guard 테스트는 항상 참입니다. 따라서 컴파일러가 이 검사를 완전히 제거할 수 있습니다!

로드맵 — Switch 문

— Switch Statements —

여러 값에 따라 분기하는 switch 문은 if-else 체인보다 훨씬 효율적인 구현이 가능합니다. 그 비결은 Jump Table입니다.

Page 32

Switch 문 예제

long switch_eg(long x, long y, long z) {
    long w = 1;
    switch(x) {
        case 1:
            w = y * z;
            break;
        case 2:
            w = y / z;
            /* Fall Through */
        case 3:
            w += z;
            break;
        case 5:
        case 6:
            w -= z;
            break;
        default:
            w = 2;
    }
    return w;
}

이 예제의 특징들:

Multiple labels: case 5와 6이 같은 코드를 공유
Fall through: case 2에서 break 없이 case 3으로 흘러감
Missing case: case 4가 없음 → default로 처리
Page 33

Jump Table 구조

Jump Table은 각 case에 대한 코드 주소를 배열처럼 저장한 테이블입니다:

Switch Form                 Jump Table          Jump Targets
switch(x) {                jtab:               Targ0: Code Block 0
  case val_0:                  Targ0
    Block 0                    Targ1            Targ1: Code Block 1
  case val_1:                  Targ2
    Block 1                    ...              Targ2: Code Block 2
  ...                          Targn-1                 ...
  case val_n-1:                                 Targn-1: Code Block n-1
    Block n-1
}

실행 방식 (Extended C로 표현):

goto *JTab[x];    // x를 인덱스로 Jump Table에서 주소를 꺼내 점프
시간 복잡도: if-else 체인은 \(O(n)\)이지만, Jump Table은 \(O(1)\)입니다! case가 아무리 많아도 테이블 인덱싱 한 번으로 바로 해당 코드로 점프합니다.
Page 34

Switch 문 — 어셈블리 Setup

switch_eg:
    movq    %rdx, %rcx        # rcx = z (rdx를 다른 용도로 쓰기 위해 이동)
    cmpq    $6, %rdi           # x:6 비교
    ja      .L8                # x > 6이면 default (.L8)로 점프
    jmp     *.L4(,%rdi,8)      # Jump Table에서 주소를 꺼내 간접 점프
레지스터용도
%rdi인자 x (switch 대상)
%rsi인자 y
%rdx → %rcx인자 z
%rax반환값
ja .L8: x가 6보다 크면 default. unsigned 비교이므로 음수도 함께 걸러냅니다 (음수는 unsigned로 보면 매우 큰 값).

jmp *.L4(,%rdi,8): .L4 + x×8 주소에서 8바이트(주소)를 읽어 그곳으로 점프합니다.

Switch 문 — Jump Table 내용

.section .rodata
  .align 8
.L4:                          # Jump Table 시작
  .quad   .L8                 # x = 0: default
  .quad   .L3                 # x = 1: case 1
  .quad   .L5                 # x = 2: case 2
  .quad   .L9                 # x = 3: case 3
  .quad   .L8                 # x = 4: default (missing case!)
  .quad   .L7                 # x = 5: case 5
  .quad   .L7                 # x = 6: case 6
인덱스 (x)점프 대상설명
0.L8case 없음 → default
1.L3case 1
2.L5case 2
3.L9case 3
4.L8case 없음 → default
5.L7case 5
6.L7case 6
빠진 case(0, 4)는 default 주소를 넣어 처리
여러 case가 같은 코드를 공유(5, 6 → .L7)하면 같은 주소를 넣음
테이블 크기 = (max_case - min_case + 1) × 8바이트

Jump Table 동작 원리

테이블 구조:

각 엔트리는 8바이트 (.quad = 64비트 주소)
베이스 주소는 .L4
총 7개 엔트리 (x = 0 ~ 6)

점프 방식:

종류어셈블리설명
Direct Jumpjmp .L8레이블 .L8로 직접 점프
Indirect Jumpjmp *.L4(,%rdi,8).L4 + x×8 주소의 내용으로 점프
간접 점프의 주소 계산:
Effective Address = .L4 + %rdi × 8
이 주소에서 8바이트를 읽음 → 그 값이 점프할 코드의 주소

예: x = 2이면 .L4 + 16 → .L5의 주소를 읽음 → .L5로 점프

Fall-Through 처리 — case 2 → case 3

case 2에는 break가 없어서 case 3으로 "떨어집니다" (fall through):

// C 코드                         // Goto 버전
case 2:                           case 2:
    w = y / z;                        w = y / z;
    /* Fall Through */                goto merge;
case 3:
    w += z;                       case 3:
    break;                            w = 1;     // 초기값

                                  merge:
                                      w += z;
Fall-Through의 의미:
case 2 진입: w = y/z 계산 → w += z 실행 (merge에서 합류)
case 3 직접 진입: w = 1 (초기값) → w += z 실행

두 경로가 merge 지점에서 합류하여 w += z를 공유합니다.
Page 39

Switch Code Block — case 2, 3 어셈블리

.L5:                            # Case 2
    movq    %rsi, %rax          # rax = y
    cqto                        # rax를 128비트로 부호 확장 (idivq 준비)
    idivq   %rcx                # rax = y / z (나눗셈)
    jmp     .L6                 # merge로 점프 (fall through 구현)

.L9:                            # Case 3
    movl    $1, %eax            # w = 1 (long w = 1의 초기값)

.L6:                            # merge:
    addq    %rcx, %rax          # w += z
    ret
핵심 포인트:
cqto (Convert Quad to Oct): %rax를 %rdx:%rax 128비트로 부호 확장. idivq가 128비트 나눗셈을 수행하기 때문에 필요
case 2: y/z 계산 후 jmp .L6으로 fall through 구현
case 3: w = 1 설정 후 자연스럽게 .L6으로 진행 (점프 불필요)
.L6에서 둘 다 w += z 실행

Switch Code Block — case 5/6, default

.L7:                            # Case 5, 6
    movl    $1, %eax            # w = 1
    subq    %rdx, %rax          # w -= z  (w = 1 - z)
    ret

.L8:                            # Default
    movl    $2, %eax            # w = 2

Case동작어셈블리
case 5, 6w = 1 - z.L7: movl $1 → subq

w = 1 초기화 이유: C 코드에서 long w = 1;이지만 w -= z는 실제로 1 - z입니다. 컴파일러가 이를 movl $1 + subq로 직접 구현합니다.
