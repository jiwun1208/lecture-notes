Great Reality #1 — 복습

이진 소수 — 퀴즈

먼저 질문: \(1011.101_2\)는 십진수로 얼마일까요?

\(1 \times 2^3 + 0 \times 2^2 + 1 \times 2^1 + 1 \times 2^0 + 1 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3}\) \(= 8 + 0 + 2 + 1 + 0.5 + 0 + 0.125 = 11.625\)

소수점 왼쪽은 \(2^0, 2^1, 2^2, \ldots\) (정수 부분), 오른쪽은 \(2^{-1}, 2^{-2}, 2^{-3}, \ldots\) (소수 부분)입니다.

Page 06

이진 소수 표현 — Fractional Binary Numbers

이진수에서 "binary point"(소수점) 오른쪽의 비트는 2의 음의 거듭제곱을 나타냅니다:

\(b_i b_{i-1} \cdots b_1 b_0 \;.\; b_{-1} b_{-2} b_{-3} \cdots b_{-j}\) \(= \sum_{k=-j}^{i} b_k \times 2^k\)

소수점 오른쪽: \(b_{-1} = 1/2\), \(b_{-2} = 1/4\), \(b_{-3} = 1/8\), ...

• \(0.111\ldots_2 = 1/2 + 1/4 + 1/8 + \cdots \to 1.0\) (무한급수). 이를 \(1.0 - \varepsilon\)으로 표기

표현 가능한 수의 한계

한계 1: 정확히 표현 가능한 수가 제한적

\(x / 2^k\) 형태만 정확히 표현됩니다. 그 외는 무한 순환소수가 됩니다:

1/3  = 0.0101010101[01]...₂   ← 무한 반복
1/5  = 0.001100110011[0011]...₂
1/10 = 0.0001100110011[0011]...₂
0.1을 이진수로 정확히 표현할 수 없습니다! 이것이 0.1 + 0.2 ≠ 0.3 문제의 근본 원인입니다.

한계 2: 고정된 소수점 위치

w비트 내에서 소수점 위치가 고정되면, 표현 가능한 수의 범위가 매우 제한됩니다. 아주 작은 수나 아주 큰 수를 동시에 표현할 수 없습니다. → 이것이 부동(floating) 소수점이 필요한 이유!

IEEE 754 — 부동소수점 표준

IEEE Standard 754 (1985년 제정)는 부동소수점 산술의 통일 표준입니다. 이전에는 제조사마다 다른 형식을 사용해 혼란이 있었습니다.

모든 주요 CPU가 이 표준을 지원합니다. 반올림, 오버플로우, 언더플로우에 대한 명확한 규칙을 제공합니다.

하드웨어에서 빠르게 구현하기 어렵지만, 수치 해석학자들의 요구가 반영된 표준입니다.

Page 11

부동소수점 표현 — 구조

부동소수점 수의 수학적 형태:

\(v = (-1)^s \times M \times 2^E\)
구성 요소의미비트 필드
s (Sign)부호: 0=양수, 1=음수최상위 1비트
M (Significand)유효숫자, 보통 \([1.0, 2.0)\) 범위frac 필드로 인코딩 (M ≠ frac)
E (Exponent)2의 지수, 스케일 결정exp 필드로 인코딩 (E ≠ exp)
  [s] [  exp  ] [       frac        ]
   1    k bits        n bits
주의: exp 필드의 값이 직접 E가 아니고, frac 필드의 값이 직접 M이 아닙니다. 인코딩 규칙이 따로 있습니다.

Normalized Values — 정규화된 값

조건: exp ≠ 000...0 이고 exp ≠ 111...1 일 때 (대부분의 수)

Exponent 인코딩:

\(E = Exp - Bias\)
\(Bias = 2^{k-1} - 1\) (k = exp 비트 수)
BiasExp 범위E 범위
Single (8bit exp)1271 ~ 254-126 ~ +127
Double (11bit exp)10231 ~ 2046-1022 ~ +1023

Significand 인코딩: 암묵적 선행 1(implied leading 1)을 가집니다:

\(M = 1.xxx\ldots x_2\) (frac 필드가 xxx...x)

frac = 000...0이면 \(M = 1.0\), frac = 111...1이면 \(M = 2.0 - \varepsilon\). "공짜로" 1비트를 더 얻는 셈입니다!

Page 14

Normalized 인코딩 예제 — 15213.0

float F = 15213.0;을 인코딩해봅시다:

15213₁₀ = 11101101101101₂
        = 1.1101101101101₂ × 2¹³

Significand:

\(M = 1.1101101101101_2\)
frac = 11011011011010000000000₂ (23비트, 소수점 뒤 부분)

Exponent:

\(E = 13, \quad Bias = 127\)
\(Exp = E + Bias = 13 + 127 = 140 = 10001100_2\) 
결과: 0  10001100  11011011011010000000000
      s    exp              frac

Denormalized Values — 비정규화된 값

조건: exp = 000...0 일 때

Normalized와 다른 두 가지 규칙:

1. Exponent: \(E = 1 - Bias\) (0 - Bias가 아님!)
2. Significand: \(M = 0.xxx\ldots x_2\) (선행 1 대신 선행 0)

Case 1: frac = 000...0 → 값 = 0

+0과 -0이 별도로 존재합니다 (s비트에 따라).

Case 2: frac ≠ 000...0 → 0에 매우 가까운 수

이 값들은 등간격(equispaced)으로 분포합니다.

Denormalized의 존재 이유: Normalized로는 0에 가까운 수를 표현할 수 없습니다. \(E = 1 - Bias\) 규칙 덕분에 Denormalized → Normalized 전환이 매끄럽게 이어집니다.

조건: exp = 111...1 일 때

expfrac의미예시
111...1000...0무한대 (Infinity)1.0/0.0 = +∞, -1.0/0.0 = -∞

부동소수점 인코딩의 시각화

수직선 위에서 부동소수점 값들의 분포:

  -∞ ← ... -Normalized ... -Denorm  -0  +0  +Denorm ... +Normalized ... → +∞
  NaN                                                                      NaN
구조 요약:
• Denormalized: 0 근처의 매우 작은 수들 (등간격)
• Normalized: 대부분의 수 (0에서 멀어질수록 간격이 넓어짐)
• ±∞: 양/음의 무한대
• NaN: 숫자가 아닌 값
Page 18

IEEE 인코딩의 특별한 성질

IEEE 부동소수점 인코딩에는 편리한 성질들이 있습니다:

1. FP 0 = 정수 0: 모든 비트가 0이면 부동소수점에서도 정수에서도 0입니다.

2. (거의) Unsigned 정수 비교 가능:
• 부호 비트를 먼저 비교한 후
• \(-0 = +0\) 처리
• NaN은 예외 (어떤 값보다 크게 나옴)
• 그 외에는 Denorm↔Norm↔∞ 순서가 unsigned 정수 비교와 일치!
이 성질 덕분에 하드웨어에서 부동소수점 비교를 정수 비교 회로로 빠르게 구현할 수 있습니다.

값의 분포 — 0 근처가 촘촘

6비트 IEEE-like 형식 (exp 3비트, frac 2비트, Bias = 3)의 수직선 분포:

  -15    -10     -5      0      5     10     15
   |------|------|------|------|------|------|
         촘촘  ←   Denorm  Norm   →  듬성듬성
핵심 관찰: 0에 가까울수록 수들이 촘촘하게 분포하고, 멀어질수록 듬성듬성해집니다. 이것이 부동소수점의 핵심 특성입니다 — 절대 오차가 아닌 상대 오차를 일정하게 유지합니다.

로드맵 — Rounding

부동소수점 연산의 핵심: "정확한 결과를 먼저 계산한 뒤, 표현 가능한 값으로 반올림한다"
Page 25

부동소수점 연산의 기본 아이디어

\(x +_f y = Round(x + y)\)
\(x \times_f y = Round(x \times y)\)

기본 절차:

1. 먼저 정확한 결과를 계산
2. 원하는 정밀도에 맞추기:
 • 지수가 너무 크면 → 오버플로우 (±∞)
 • frac에 맞지 않으면 → 반올림(Round)

부동소수점 덧셈

\((-1)^{s_1} M_1 \cdot 2^{E_1} + (-1)^{s_2} M_2 \cdot 2^{E_2}\) (\(E_1 \geq E_2\) 가정)

절차:
1. 정렬(Align): 작은 지수의 수를 오른쪽 시프트하여 binary point를 맞춤
2. 덧셈: Significand끼리 더함
3. 정규화: \(M \geq 2\)이면 오른쪽 시프트 & E 증가, \(M < 1\)이면 왼쪽 시프트 & E 감소
4. 오버플로우 체크: E가 범위 초과하면 ±∞
5. 반올림: M을 frac 정밀도에 맞춤
정렬 과정에서 작은 수의 하위 비트가 사라질 수 있습니다. 이것이 1e20 + 3.14 = 1e20이 되는 이유입니다.

Page 31

부동소수점 곱셈

\((-1)^{s_1} M_1 \cdot 2^{E_1} \times (-1)^{s_2} M_2 \cdot 2^{E_2}\)

부호: \(s = s_1 \oplus s_2\) (XOR)
유효숫자: \(M = M_1 \times M_2\)
지수: \(E = E_1 + E_2\)

정규화 후 반올림합니다. \(M \geq 2\)이면 오른쪽 시프트 & E 증가.

요약 — Floating Point

IEEE Floating Point 핵심 정리:

• \(v = (-1)^s \times M \times 2^E\) 형태로 수를 표현
• 명확한 수학적 성질: "정확한 결과를 계산 후 반올림"으로 동작 이해 가능
• 구현과 독립적으로 동작을 추론할 수 있음
실수 산술과 다른 점:
• 결합법칙 불성립 — 덧셈과 곱셈 모두
• 분배법칙 불성립 — 곱셈
• 컴파일러와 수치 계산 프로그래머에게 큰 도전

이 챕터의 핵심 교훈: 부동소수점은 근사값이며, 수학의 실수와 동일하게 취급하면 안 됩니다.
