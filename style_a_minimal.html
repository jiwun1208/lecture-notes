파이프라이닝 (Pipelining) 개요

파이프라이닝은 여러 명령어를 동시에 실행하는 기법으로, 각 명령어는 서로 다른 실행 단계에 있으면서 동시에 처리된다. 세탁기-건조기 비유로 이해하면 쉽다.
상세 설명

파이프라이닝의 핵심 아이디어는 겹침(overlap)이다. 한 명령어가 실행 단계에 있을 때, 다음 명령어는 해독 단계에, 그 다음 명령어는 인출 단계에 있을 수 있다.


세탁에 비유하면: 첫 번째 빨래가 건조기에 들어간 순간 두 번째 빨래를 세탁기에 넣을 수 있다. 이렇게 하면 전체 시간이 크게 단축된다.


중요한 점은 개별 명령어의 실행 시간은 줄어들지 않는다는 것이다. 줄어드는 것은 단위 시간당 처리량(throughput)이다.

핵심 용어
영어한국어설명
Pipelining 파이프라이닝 명령어 실행 단계를 겹쳐서 처리하는 기법
Throughput 처리량 단위 시간당 완료되는 명령어 수
Latency 지연 시간 하나의 명령어가 시작부터 끝까지 걸리는 시간
시험 포인트: "파이프라이닝은 latency를 줄이는 것이 아니라 throughput을 높이는 것이다"는 거의 매년 출제된다.

5단계 RISC 파이프라인

MIPS 프로세서의 고전적인 5단계: IF(명령어 인출) → ID(해독) → EX(실행) → MEM(메모리 접근) → WB(레지스터 쓰기)
상세 설명

각 단계는 한 클럭 사이클을 차지한다. 이상적인 경우 5개의 명령어가 동시에 각기 다른 단계에서 실행되므로, CPI(Cycles Per Instruction)가 이론적으로 1에 수렴한다.


실제로는 해저드(hazard) 때문에 스톨(stall)이 발생하여 CPI가 1보다 커진다.

클럭: |  1  |  2  |  3  |  4  |  5  |  6  |  7  |
I1:   | IF  | ID  | EX  | MEM | WB  |     |     |
I2:   |     | IF  | ID  | EX  | MEM | WB  |     |
I3:   |     |     | IF  | ID  | EX  | MEM | WB  |
핵심 용어
영어한국어설명
IF (Instruction Fetch) 명령어 인출 메모리에서 다음 명령어를 가져오는 단계
ID (Instruction Decode) 명령어 해독 명령어를 분석하고 레지스터 값을 읽는 단계
EX (Execute) 실행 ALU 연산 수행 단계

파이프라인 해저드와 해결법

해저드는 다음 명령어를 예정대로 실행할 수 없게 만드는 상황이다. 구조적(Structural), 데이터(Data), 제어(Control) 해저드 3종류가 있다.
상세 설명

1. 구조적 해저드 (Structural Hazard)
하드웨어 자원이 부족해서 발생한다. 예: 메모리가 하나뿐인데 IF와 MEM이 동시에 접근하려 할 때.


2. 데이터 해저드 (Data Hazard)
이전 명령어의 결과를 다음 명령어가 필요로 할 때 발생한다. 포워딩(forwarding)으로 대부분 해결 가능하다.


3. 제어 해저드 (Control Hazard)
분기(branch) 명령어 때문에 다음에 실행할 명령어가 불확실할 때 발생한다. 분기 예측(branch prediction)으로 완화한다.

포워딩(바이패싱)은 데이터 해저드의 가장 대표적인 해결책이다. EX 단계의 결과를 다음 명령어의 EX 입력으로 바로 넘기는 것. 그림으로 그릴 수 있어야 한다.
